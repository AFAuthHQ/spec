# AFAuth v0.2 — Implementation Scope & Roadmap

> Supersedes [`v0.1-scope.md`](v0.1-scope.md). Bridges the normative spec
> (`spec/core.md`, including the optional §10 attestation layer, and
> `spec/directory.md`) to the v0.2 reference implementation.
>
> **The wire protocol is unchanged: discovery documents still declare
> `afauth_version: "0.1"`.** v0.2 is an *implementation* milestone that ships
> the optional attestation layer and the service directory — the explicit v0.1
> non-goals — plus the ergonomics that make attested signup the happy path. It
> is not a breaking protocol change.

## Goal

Build on the v0.1 reference (per-request Ed25519 signing, two-step claim, key
rotation, owner-initiated revocation) with the discovery and spam-resistance
layers the ecosystem needs:

- a hosted **service directory** so agents can discover AFAuth services without
  a central gatekeeper (federation-friendly read API);
- a hosted **trust attestor** so a service can require a human-backed,
  privacy-preserving attestation (AFAP-0006) without operating its own;
- SDK + CLI ergonomics so that requiring attestation — and minting one — is a
  one-liner, not a project.

The bar remains **reference quality**, not production. Per-service production
polish (observability, alerting, audit pipelines) is still deferred.

## Delivered in v0.2

### Spec additions (accepted proposals)

- **AFAP-0003 — Service directory.** Normative listing protocol in
  [`spec/directory.md`](../spec/directory.md); listing schema at
  [`schemas/listing.json`](../schemas/listing.json).
- **AFAP-0006 — `afauth-trust` attestor.** Optional agent attestation
  ([`spec/core.md` §10](../spec/core.md)): the `afauth-trust` issuer binds an
  agent's account DID to a human-controlled account and mints short-lived,
  audience-bound JWTs verified offline against a published JWKS. Adds the
  pairwise human pseudonym `sub_h` (§10.4) and one-active-binding-per-agent
  semantics (§10.5), plus the §C.7 `trust-attestation` conformance vectors.
- **AFAP-0007 — Recognized-attestor refresh.** Updates the §10.3 recognized
  identifier list against 2026 deployed reality (drops `aws-iam-agent` and
  `stripe-projects`, which have no externally verifiable assertion; adds
  `visa-trusted-agent`).

### Reference services (new repos)

| Artefact | Repo | Purpose |
|---|---|---|
| Registry | `AFAuthHQ/registry` | Hosted service directory at `registry.afauth.org`. One-time host-control challenge + proof, listing create/update/soft-delete, unauthenticated CORS-open read API with cursor pagination and `updated_since` so mirrors can converge (AFAP-0003). |
| Trust attestor | `AFAuthHQ/trust` | Hosted `afauth-trust` at `trust.afauth.org`. Human link ceremony (loopback-callback friendly), JWKS, per-service attestation JWTs, `sub_h` pairwise pseudonyms, Google sign-in, human-facing binding management, signing-key rotation (AFAP-0006). |

Both are independent deployments; the directory is federation-friendly (the
schema and read contract are public) and the attestor is one of several the
spec recognizes — neither is a required dependency of the core protocol.

### SDK (npm, pnpm workspace)

| Package | Version | v0.2 surface |
|---|---|---|
| `@afauthhq/core` | `0.1.0` | Unchanged — shared `did:key` codec, RFC 9421 canonicalisation, content-digest, error envelope, types. |
| `@afauthhq/agent` | `0.2.0` | Adds `TrustClient` — the agent-side link + token client for `afauth-trust`. |
| `@afauthhq/server` | `0.3.0` | Adds attestation verification (`trustAttestor()`, `JwksAttestor`, `HmacAttestor`, `MultiAttestor`); the `defineService` factory (wires `attestation: "required"` + discovery `unclaimed_mode: "attested_only"` + bundled `trustAttestor()` by default); `sweepExpiredAccounts`. |

The agent SDK is **Node 20+** (up from Node 18 in v0.1).

### CLI

`afauth` **v0.4.0** adds the `afauth trust` subcommand (`link`, `token`,
`status`, `forget`); `afauth signup` auto-mints an audience-bound attestation
from the cached binding when a service's discovery doc requires one
(`attested_only`). Binding state is stored at `~/.afauth/trust.json` (chmod 600).

## Non-goals (v0.2) — still deferred

- **`did:web` resolution.** The reference SDK and Worker still resolve
  `did:key` only, in-process ([ADR-0003](adr/0003-did-key-only-resolver.md)).
  The resolver hook remains opt-in; `did:web` is recognised in the types.
- **Recipient handlers beyond `email`.** `phone`, `oidc`, and `did` remain
  interface-only in the SDK; the reference Worker ships `email` only.
- **Refresh tokens / long-lived agent sessions.** AFAuth still signs every
  request; token-based auth remains out of scope and proposal-tracked.
- **Browser SDK.** The agent SDK is Node-only.
- **x402 payment binding** (AFAP-0004, *Draft*) — advertisable in discovery,
  but no reference enforcement path ships.
- **Trust attestor breadth:** OAuth providers beyond Google, and
  payment-method verification *issuance* (the `payment` verification value is
  scaffolded in the data model but not issued).
- **Performance benchmarks.**

## Conformance

The §C.1–§C.6 corpus and its gates are unchanged from v0.1 — a v0.1 agent or
service that ignores attestation interoperates unchanged. v0.2 adds the
**optional** §C.7 `trust-attestation` vectors (`vectors/trust-attestation/`):
well-formed, runtime-only (no `sub_h`), and malformed-`sub_h` cases. They apply
only to implementations that consume `afauth-trust` attestations.

## Status

- `AFAuthHQ/registry` — live at `registry.afauth.org` (AFAP-0003).
- `AFAuthHQ/trust` — live at `trust.afauth.org` (AFAP-0006/0007).
- `AFAuthHQ/typescript-sdk` — `@afauthhq/{core@0.1.0, agent@0.2.0,
  server@0.3.0, worker@0.3.0}` published to npm.
- `AFAuthHQ/cli` — `afauth` tagged `v0.4.0`; Homebrew + `install.sh` + Go
  install all resolve to the latest release.

## Next concrete actions (post-v0.2)

1. **`did:web` resolver** — promote the opt-in hook to a shipped, cached
   resolver; pair with a conformance vector set.
2. **Refresh-token / session semantics** — draft the proposal the v0.1/v0.2
   non-goal defers.
3. **Additional recipient handlers** — at least one of `oidc` / `phone` as a
   reference implementation, with §C.4-aligned vectors exercised end-to-end.
